# 决策记录与工作日志

## 决策记录

### [DEC-001] - 2026-03-30 - 示例优先保证零依赖可运行

**背景**
示例 Skill 的第一目标是教学。如果一开始就依赖外部 API 或密钥，新人很难在 5 分钟内跑通。

**选项**
1. 直接接真实天气 API
2. 先用静态样例数据构建可运行示例

**决策**
选择选项 2。

**理由**
先保证任何人复制后都能成功运行，再把真实 API 接入作为下一阶段任务。

**影响**
当前输出是演示数据，但目录结构、文档体系和脚本组织都可以直接复用。

### [DEC-002] - 2026-03-30 - `.env.example` 与 `assets/config.yaml.example` 分开

**背景**
环境变量模板和结构化配置模板的读取方式不同，混在一起容易让维护者误解。

**选项**
1. 所有模板都放 `assets/`
2. `.env.example` 放根目录，结构化配置模板放 `assets/`

**决策**
选择选项 2。

**理由**
这样更接近常见仓库实践，也更容易让用户直接复制使用。

**影响**
Starter 模板和示例 Skill 都按这个结构展示。
