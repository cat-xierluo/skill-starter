# 什么是 GitHub？

很多人第一次看到 GitHub，会在心里默默发音——"gayhub"？

其实它叫 **GitHub**（git-hub），和"gay"没有任何关系。它是程序员的"云端基地"，全宇宙最大的代码托管平台。

---

## GitHub 像什么？

| 现实中的东西 | GitHub 里对应的概念 |
|---|---|
| 云硬盘 | 存代码的在线空间 |
| 朋友圈 | 分享代码、看到别人的动态 |
| 版本历史（iCloud/Google Docs的"查看修订历史"） | 自动记录每一次修改 |
| 多人协作写文档 | 多人一起开发同一个项目 |

简单说：**GitHub ≈ 云盘 + 微信朋友圈 + 修订历史 + 协作工具**

---

## 为什么我们需要 GitHub？

**1. 代码不再丢失**
本地写代码，电脑坏了就全没了。GitHub 把代码存在云端，永久保存。

**2. 多人协作变得简单**
A 写前端，B 写后端，C 做测试——大家在不同地方，同时改代码，GitHub 自动合并。

**3. 记录每一次修改**
就像 iCloud 记录文档的每个版本，GitHub 记录你每次改了什么、谁改的、为什么改。

**4. 展示你的作品**
面试时给 HR 看 GitHub 主页，比简历更有说服力。

---

## 核心概念

### Repository（仓库）
项目的"文件夹"。一个 App、一个网站、一本书，都可以是一个 Repository。

> 就像你电脑里的一个文件夹，里面装着这个项目所有的代码和文件。

### Commit（提交）
每次"保存进度"就是一个 Commit。你改了什么、为什么改，都会被记录。

> 就像游戏里的存档点，随时可以回到这个版本。

### Branch（分支）
从主版本分出来的"平行宇宙"。你在分支上做实验，不影响原版。

> 就像你在文档里"另存为"一个新文件，原文件保持不变。

### Pull Request（PR，请求拉取）
你改好了，想把自己的修改合并回主版本，就发一个 PR，等待别人审核通过。

> 就像你修改了文档后，发给同事审批，同事点了"同意"才算正式合并。

---

## 一句话总结

**GitHub 就是程序员的云硬盘 + 协作平台 + 版本历史本。**

不管你是想保存代码、跟别人合作、还是展示作品，GitHub 都是你的第一站。
