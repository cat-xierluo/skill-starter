# 常用术语表

用最简单的话，解释程序员和 AI 圈的高频词汇。

---

## A

### Agent
**中文：** 智能体 / AI 助手
**解释：** 能帮你做事的人工智能程序。你说"帮我查天气"，它就真的去查了。
**例：** "我的 AI 助手 Agent 帮我订了机票。"

### API
**中文：** 接口
**解释：** 两个软件之间对话的"窗口"。你想让 AI 帮你查天气，AI 通过 API 问天气服务要数据。
**例：** "这个 App 通过 API 连接到天气预报系统。"

### Artifact
**中文：** 工件 / 产物
**解释：** AI 生成的东西，比如一篇文章、一段代码、一张图片。
**例：** "这段代码是 AI 的 artifact，我可以复制使用。"

---

## B

### Branch
**中文：** 分支
**解释：** 从项目主线上分出来的一条独立工作线，用来实验新功能，不影响原版。
**例：** "我在 new-feature 分支上做实验，做完了再合并回去。"

---

## C

### CLI（Command Line Interface）
**中文：** 命令行界面
**解释：** 通过打字来操作电脑的方式，不像鼠标点击那样直观，但效率更高。
**例：** "在终端输入 `npm install`，用 CLI 安装了一个工具。"

### Clone
**中文：** 克隆 / 复制
**解释：** 把 GitHub 上的项目下载到自己的电脑本地。
**例：** "我 clone 了这个仓库到本地，开始改代码。"

### Commit
**中文：** 提交
**解释：** 每一次保存并记录改了什么，就是一个 Commit。附带一条说明。
**例：** "刚才做了 Commit，备注是'修复了登录按钮不显示的bug'。"

### Config
**中文：** 配置文件
**解释：** 存放软件设置的文件。比如主题颜色、接口地址、功能开关。
**例：** "我把 API 地址改成了测试环境，在 config 文件里。"

---

## F

### Fork
**中文：** 派生 / 复刻
**解释：** 把别人的项目复制一份到自己账号下，成为独立副本，可以随便改。
**例：** "我 fork 了这个开源项目，加了中文翻译。"

---

## G

### Git
**中文：** Git（版本控制工具）
**解释：** 记录代码修改历史的工具，让你随时可以回到任意版本。
**例：** "我用了 Git，每次改代码都有记录，不怕改坏了。"

### GitHub
**中文：** GitHub（代码托管平台）
**解释：** 托管 Git 仓库的网站，程序员的"云硬盘 + 朋友圈"。
**例：** "这个项目我已经上传到 GitHub 了，你去看看。"

---

## I

### IDE（Integrated Development Environment）
**中文：** 集成开发环境
**解释：** 写代码的软件，比如 VS Code。集编辑、运行、调试于一身。
**例：** "我用 VS Code 这个 IDE 写 Python。"

---

## L

### LLM（Large Language Model）
**中文：** 大语言模型
**解释：** 能理解和生成文字的 AI 模型，像 ChatGPT、Claude 就是 LLM。
**例：** "这个功能用的是 GPT-4，是 OpenAI 的 LLM。"

---

## M

### Markdown
**中文：** 轻量级标记语言
**解释：** 用简单符号排版的写作格式，`# 标题`、`**粗体**`，写出来比 Word 更干净。
**例：** "这份文档是用 Markdown 写的，比 Word 轻多了。"

---

## O

### Open Source
**中文：** 开源
**解释：** 代码公开，任何人都能看、用、改进的software。
**例：** "React 是 Facebook 的开源项目，全世界都在用。"

---

## P

### PR（Pull Request）
**中文：** 拉取请求
**解释：** 你改好了代码，想把自己的修改合入主项目，就发一个 PR，等待审核。
**例：** "我提了一个 PR，等同事 review 通过就能合并了。"

### Push
**中文：** 推送
**解释：** 把本地写的代码上传到 GitHub。
**例：** "我 push 了三次代码到远程仓库。"

### Pull
**中文：** 拉取
**解释：** 把 GitHub 上的最新代码下载到本地。
**例：** "你先 pull 一下，我刚更新了很多内容。"

---

## R

### Repo（Repository）
**中文：** 仓库
**解释：** 一个项目的所有代码和文件存放的地方。
**例：** "这个 Repo 有 3000 颗 Star，说明很受欢迎。"

---

## S

### SDK（Software Development Kit）
**中文：** 软件开发包
**解释：** 别人写好的一整套工具，拿来就能用，省得自己从头造轮子。
**例：** "用了地图 SDK，我不用自己写定位功能，地图公司已经封装好了。"

### SSH（Secure Shell）
**中文：** 安全外壳协议
**解释：** 一条加密的"安全通道"，让你安全地远程连接服务器。
**例：** "我用 SSH 连接到服务器，在上面部署了网站。"

### Star
**中文：** 星标 / 收藏
**解释：** 给喜欢的项目点个赞，也相当于收藏。
**例：** "这个项目有 10000 颗 Star，非常火！"

---

## T

### Token
**中文：** 令牌 / 密钥
**解释：** 使用 AI 或 API 的"通行证"，证明你有权限用它。
**例：** "我的 API Token 泄露了，赶紧换一个。"

---

## V

### Variable
**中文：** 变量
**解释：** 给数据起的名字。就像把东西放进盒子里，盒子上的标签就是变量名。
**例：** "我设了一个变量 `username = "小明"`，后面直接用这个名字就行。"

---

## W

### Webhook
**中文：** 网络钩子
**解释：** 当某个事件发生时，自动触发一个动作。比如"代码 push 到 GitHub 时，自动部署网站"。
**例：** "我设置了 Webhook，每次 PR 合并时自动运行测试。"

---

## 一句话背完

> **Agent** 帮你做事，**API** 让软件对话，**CLI** 打字控制电脑，**Git** 记录历史，**GitHub** 云端托管，**Repo** 存放代码，**PR** 请求合并，**Push/Pull** 上下传代码，**Fork** 复制项目，**SDK** 省事工具包，**Token** 是通行证，**LLM** 是大语言模型。

记住这些，你已经比 80% 的人更懂行了！
